### Task 2: Enhance Executions Stats
**File**: `apps/backend/src/executions/executions.service.ts`
- Modify `getStats()` method to query additional metrics:
  - `totalExecutions`: count all executions
  - `successfulExecutions`: count where status = 'SUCCESS'
  - `failedExecutions`: count where status = 'FAILED'
  - `successRate`: `(successfulExecutions / totalExecutions * 100)` rounded to 1 decimal, default 0
  - `avgDuration`: average of `(completedAt - startedAt)` in milliseconds for completed executions
  - `executions24h`: count where `startedAt >= now - 24 hours`
  - `activeWorkflows`: count workflows where `isActive = true`
- Use Prisma `aggregate` and `groupBy` for efficient queries
- Return all fields as a typed `DashboardStats` interface

### Task 3: Add Execution Filters to Service
**File**: `apps/backend/src/executions/executions.service.ts`
- Modify `findAll()` to accept `ExecutionFilterDto`
- Build Prisma `where` clause dynamically:
  ```
  where: {
    ...(workflowId && { workflowId }),
    ...(status && { status }),
    ...(dateFrom || dateTo) && {
      startedAt: {
        ...(dateFrom && { gte: new Date(dateFrom) }),
        ...(dateTo && { lte: new Date(dateTo) }),
      }
    }
  }
  ```
- Include pagination with `skip` and `take` from page/limit
- Return `{ data: Execution[], total: number, page: number, limit: number }`

- Use `ValidationPipe` with `transform: true` for query param type coercion

### Task 6: Enrich Workflows with Execution Stats
**File**: `apps/backend/src/workflows/workflows.service.ts`
- Modify `findAll()` to accept `WorkflowFilterDto`
- Add `search` filter: Prisma `where: { OR: [{ name: { contains: search } }, { description: { contains: search } }] }`
- Add `status` filter: `isActive: status === 'active' ? true : status === 'inactive' ? false : undefined`
- For each workflow in results, compute:
  - `lastExecution`: most recent execution's `startedAt` and `status`
  - `successRate`: percentage of SUCCESS executions out of total for that workflow
  - `executionCount`: total executions for that workflow
- Use a single Prisma query with `include: { executions: { select: { status: true, startedAt: true }, orderBy: { startedAt: 'desc' } } }` then compute stats in-memory
- OR use `_count` and `groupBy` for better performance on large datasets
