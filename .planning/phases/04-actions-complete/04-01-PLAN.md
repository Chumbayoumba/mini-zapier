# Plan 04-01: Template Engine + Data Flow

## Metadata
- **Phase:** 04-actions-complete
- **Wave:** 1
- **Depends on:** none
- **Estimated scope:** 3 files (1 new utility, 1 new spec, 1 modify)

## Objective
Create a shared `TemplateEngine` utility that resolves `{{steps.nodeId.output.field}}` and `{{trigger.field}}` expressions in action configs. Integrate it into `EngineService.executeStepWithRetry()` so that ALL string values in `node.data.config` are interpolated BEFORE being passed to action handlers. This eliminates the need for per-action interpolation (like Telegram's private `interpolateMessage`) and enables ACT-06 data flow between steps.

## Tasks

### Task 1: Create TemplateEngine utility
**File:** `apps/backend/src/engine/template-engine.ts`
**Action:** Create
**Details:**
- Export class `TemplateEngine` with static methods (no DI needed ‚Äî pure utility)
- `static resolve(template: string, context: TemplateContext): string` ‚Äî resolves all `{{...}}` expressions in a single string
- `static resolveConfig(config: Record<string, any>, context: TemplateContext): Record<string, any>` ‚Äî deep-walks a config object, resolving all string values. Preserves non-string values (numbers, booleans, objects). Handles nested objects and arrays recursively.
- `static resolveValue(path: string, context: TemplateContext): any` ‚Äî resolves a dot-path like `steps.node1.output.data.name` against context
- Export interface `TemplateContext`:
  ```typescript
  export interface TemplateContext {
    steps: Record<string, any>;   // nodeId ‚Üí step output
    trigger: any;                  // trigger data
    env?: Record<string, string>;  // future: environment variables
  }
  ```
- **Template syntax supported:**
  - `{{steps.nodeId.output.field}}` ‚Äî access previous step output
  - `{{steps.nodeId.output.items[0].name}}` ‚Äî array index access
  - `{{trigger.field}}` ‚Äî access trigger data
  - `{{trigger.body.nested.path}}` ‚Äî nested trigger paths
- **Path resolution logic** in `resolveValue`:
  1. Split path by `.` but respect array brackets: `items[0]` ‚Üí navigate to `items` then index `0`
  2. Walk the context object following each segment
  3. For `[N]` segments, parse N as integer and index into array
  4. If any segment resolves to `undefined` or `null`, return the original `{{expression}}` as-is (graceful fallback)
  5. If final value is an object/array, return `JSON.stringify(value)` for string interpolation, or return raw value if the entire template is a single expression (e.g., config value IS `{{steps.node1.output}}` with no surrounding text)
- **Edge cases:**
  - Empty string ‚Üí return empty string (no processing)
  - No `{{` in string ‚Üí return string as-is (fast path)
  - Malformed `{{` without `}}` ‚Üí return as-is (no crash)
  - Nested `{{` ‚Üí treat as literal (no recursive resolution)
  - Value is `undefined`/`null` ‚Üí keep original `{{expression}}` placeholder
  - Value is `0` or `false` ‚Üí convert to string `"0"` / `"false"` (truthy check trap)

### Task 2: Integrate TemplateEngine into EngineService
**File:** `apps/backend/src/engine/engine.service.ts`
**Action:** Modify
**Details:**
- Import `TemplateEngine` and `TemplateContext` from `./template-engine`
- In `executeStepWithRetry()`, AFTER constructing the `input` object (line ~287-294) and BEFORE calling `this.executeAction()`:
  ```typescript
  // Build template context from execution state
  const templateContext: TemplateContext = {
    steps: context.stepResults,
    trigger: context.triggerData || {},
  };
  // Resolve {{...}} expressions in all config string values
  const resolvedConfig = TemplateEngine.resolveConfig(
    node.data.config || {},
    templateContext,
  );
  const input = {
    ...resolvedConfig,
    _context: {
      ...context.stepResults,
      triggerData: context.triggerData,
      integrations: context.integrations,
    },
  };
  ```
- This means actions receive ALREADY-RESOLVED config values. They no longer need to implement their own interpolation.
- The `_context` field is still passed for backward compatibility (actions that manually access `_context.triggerData` etc. continue working).
- Also update `stepLog` input to log the resolved config (not raw) for debugging:
  ```typescript
  // In step log creation, change input to resolvedConfig
  input: resolvedConfig,
  ```

### Task 3: Create comprehensive TemplateEngine tests
**File:** `apps/backend/src/engine/template-engine.spec.ts`
**Action:** Create
**Details:**
- Test groups:
  1. **`resolve()` ‚Äî basic interpolation:**
     - `"Hello {{trigger.name}}"` + `{trigger: {name: "World"}}` ‚Üí `"Hello World"`
     - `"{{steps.http1.output.data.id}}"` + context ‚Üí resolved value
     - Multiple expressions in one string: `"{{trigger.first}} {{trigger.last}}"` ‚Üí both resolved
     - No expressions: `"plain text"` ‚Üí `"plain text"`
  2. **`resolve()` ‚Äî nested paths:**
     - `"{{steps.node1.output.items[0].name}}"` ‚Äî array index
     - `"{{trigger.body.deeply.nested.value}}"` ‚Äî deep nesting
  3. **`resolve()` ‚Äî graceful fallback:**
     - Missing path: `"{{steps.missing.output}}"` ‚Üí `"{{steps.missing.output}}"` (kept as-is)
     - Partial path: `"{{steps.node1.output.nonexistent.deep}}"` ‚Üí kept as-is
     - `undefined` trigger: `"{{trigger.missing}}"` ‚Üí kept as-is
  4. **`resolve()` ‚Äî edge cases:**
     - Empty string ‚Üí `""`
     - Falsy values: `{{trigger.count}}` where count=0 ‚Üí `"0"`
     - Boolean: `{{trigger.active}}` where active=false ‚Üí `"false"`
     - Object as value in mixed string ‚Üí JSON.stringify
     - Single expression as entire string: `"{{steps.node1.output}}"` ‚Üí returns raw value (object/array, not stringified)
     - Malformed: `"{{broken"` ‚Üí `"{{broken"` (untouched)
  5. **`resolveConfig()` ‚Äî deep object walk:**
     - Simple flat config: `{url: "{{trigger.url}}", method: "GET"}` ‚Üí url resolved, method untouched
     - Nested config: `{headers: {Authorization: "Bearer {{steps.auth.output.token}}"}}` ‚Üí nested resolved
     - Array in config: `{recipients: ["{{trigger.email}}", "admin@test.com"]}` ‚Üí first resolved
     - Non-string values preserved: `{timeout: 5000, retries: 3}` ‚Üí untouched
     - Mixed: `{body: {name: "{{trigger.name}}", count: 42}}` ‚Üí name resolved, count untouched
  6. **`resolveValue()` ‚Äî path resolution:**
     - Simple path: `"trigger.name"` ‚Üí value
     - Array index: `"steps.node1.output.items[0]"` ‚Üí first item
     - Out-of-bounds array: `"steps.node1.output.items[99]"` ‚Üí undefined
     - Null in path: `"steps.node1.output"` where output is null ‚Üí null
