### Task 1: Harden HTTP Request Action
**File:** `apps/backend/src/engine/actions/http-request.action.ts`
**Action:** Modify
**Details:**
- Add response size limit: configure `maxContentLength` and `maxBodyLength` on the axios instance (default 10MB = `10 * 1024 * 1024`). Accept optional `maxResponseSize` from config.
  ```typescript
  const maxResponseSize = config.maxResponseSize || 10 * 1024 * 1024;
  const client = axios.create({
    timeout,
    maxRedirects: 0,
    maxContentLength: maxResponseSize,
    maxBodyLength: maxResponseSize,
  });
  ```
- Add input validation at the start of `execute()`:
  ```typescript
  if (!url || typeof url !== 'string') {
    throw new BadRequestException('URL is required and must be a string');
  }
  const validMethods = ['GET', 'POST', 'PUT', 'DELETE', 'PATCH', 'HEAD'];
  const normalizedMethod = (method || 'GET').toUpperCase();
  if (!validMethods.includes(normalizedMethod)) {
    throw new BadRequestException(`Invalid HTTP method: ${method}. Allowed: ${validMethods.join(', ')}`);
  }
  ```
- Parse `headers` if it's a string (user may paste JSON from frontend textarea):
  ```typescript
  let parsedHeaders = headers || {};
  if (typeof parsedHeaders === 'string') {
    try { parsedHeaders = JSON.parse(parsedHeaders); }
    catch { throw new BadRequestException('Headers must be valid JSON'); }
  }
  ```
- Parse `body` if it's a string and method supports body:
  ```typescript
  let parsedBody = body;
  if (typeof parsedBody === 'string' && parsedBody.trim()) {
    try { parsedBody = JSON.parse(parsedBody); }
    catch { /* keep as string — might be plain text */ }
  }
  ```
- Note: Template interpolation of URL/headers/body is now handled by TemplateEngine in EngineService (Plan 04-01). No interpolation code needed here.

### Task 3: Harden Email Action
**File:** `apps/backend/src/engine/actions/email.action.ts`
**Action:** Modify
**Details:**
- Add per-workflow SMTP support: Instead of creating transporter in constructor, create it per-execution based on config or fallback to env:
  ```typescript
  // Remove transporter from constructor
  // In execute():
  private createTransporter(config: any) {
    // Priority: per-step config → _context.integrations.email → global env
    const smtpConfig = config.smtp || config._context?.integrations?.email || {
      host: this.configService.get('SMTP_HOST'),
      port: this.configService.get('SMTP_PORT'),
      user: this.configService.get('SMTP_USER'),
      pass: this.configService.get('SMTP_PASSWORD'),
    };
    return nodemailer.createTransport({
      host: smtpConfig.host,
      port: Number(smtpConfig.port) || 587,
      secure: smtpConfig.secure ?? (Number(smtpConfig.port) === 465),
      auth: {
        user: smtpConfig.user,
        pass: smtpConfig.pass,
      },
    });
  }
  ```
- Add input validation:
  ```typescript
  if (!to || typeof to !== 'string') {
    throw new BadRequestException('Recipient email address (to) is required');
  }
  // Simple email regex
  const emailRegex = /^[^\s@]+@[^\s@]+\.[^\s@]+$/;
  const recipients = to.split(',').map(e => e.trim());
  for (const email of recipients) {
    if (!emailRegex.test(email)) {
      throw new BadRequestException(`Invalid email address: ${email}`);
    }
  }
  if (!subject || typeof subject !== 'string') {
    throw new BadRequestException('Email subject is required');
  }
  ```
- Add CC/BCC support:
  ```typescript
  const { to, subject, body, isHtml = false, cc, bcc, from } = config;
  // In sendMail:
  const mailOptions: any = {
    from: from || smtpConfig.user || this.configService.get('SMTP_USER'),
    to: recipients.join(', '),
    subject,
    [isHtml ? 'html' : 'text']: body || '',
  };
  if (cc) mailOptions.cc = cc;
  if (bcc) mailOptions.bcc = bcc;
  ```
- Import `BadRequestException` from `@nestjs/common`
- Remove the old constructor transporter initialization; use `createTransporter()` method in `execute()`
