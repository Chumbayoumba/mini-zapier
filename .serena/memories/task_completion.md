When a task is completed, run the existing validation commands relevant to the touched area. At minimum use the repo-level scripts `pnpm build`, `pnpm test`, and `pnpm lint` when changes affect code broadly; use package-level scripts if only backend or frontend changed. For Prisma/data layer changes, also run the existing migrate/seed commands as needed. For deployment or runtime behavior, verify with real HTTP/UI flows after build/test rather than relying only on static inspection.
