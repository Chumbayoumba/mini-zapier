Project code is TypeScript-first across frontend and backend. Prettier at repo root enforces single quotes, semicolons, trailing commas, printWidth 100, tabWidth 2, LF line endings. Backend uses ESLint with @typescript-eslint recommended; explicit return types are not required, no-explicit-any is warning-level, and unused vars are warn-level with underscore-prefixed args ignored. Frontend uses Next.js core-web-vitals and TypeScript ESLint config. Follow existing monorepo patterns, keep code in English, and prefer strict typed DTOs/hooks/services over loose casts.
